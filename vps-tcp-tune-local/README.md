# VPS TCP Tune 本地分类版

这是 `Eric86777/vps-tcp-tune` 的本地化分类版本，作为独立目录加入 LTOOLS。现有根目录 `ltools.sh` 未被修改，也不会自动进入本套件。

## 使用

```bash
cd vps-tcp-tune-local
sudo bash run.sh
```

也可以直接进入某个分类：

```bash
sudo bash categories/02-network-optimization.sh
```

## 结构

```text
vps-tcp-tune-local/
├── run.sh                       分类总入口
├── lib/common.sh                本地动作调用与分类菜单
├── categories/                  11 个独立分类入口
└── upstream/
    ├── net-tcp-tune.sh          本地共享核心
    ├── snell-upgrade-patch.sh   本地 Snell 修复脚本
    └── LICENSE                  上游 MIT 许可证
```

分类入口不复制核心函数，而是通过本地参数 `--action <上游编号>` 调用共享核心。这种方式保留上游功能依赖关系，同时让入口按品类拆分。

## 本地化边界

- 启动分类和核心功能时，不会在线下载 `Eric86777/vps-tcp-tune` 的主脚本。
- 上游自带的第三方工具、内核包、IP 数据及软件安装源仍可能按原功能设计访问外部网络。
- 核心中的两处上游名称正则仅用于清理旧版在线别名，不会发起网络请求。
- 本地快照不会自动跟随上游更新；升级前应重新审计并测试差异。

## 风险提示

这些功能可更换内核、修改 sysctl、防火墙、DNS、代理服务和软件包。请使用 VPS 快照，并确保能够通过服务商控制台恢复。
